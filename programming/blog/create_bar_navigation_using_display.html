Website Navigation

What is it?

Website navigation is one of the major factors for websites, they have a primary purpose for moving their visitors from one page to another. Without having the navigation, it would be difficult for website visitors to adapt the interest in browsing with and becomes more frustrating for them to find pages they want to visit.

Constructing website navigations has to be planned out well before building, we have to consider why and how to want our navigation to interact. Most importantly, we need to consider how we would design to attract interest to our target audiences.

How does it work?

When websites are visited, visitors often see a navigation across every page in different ways and approaches to directing them to a specific page. Those may be seen to be Bar Navigation, Side Navigation, Secondary Navigation and Footer Navigation. They all have different purposes on how they are used to interact with the visitor.

Navigation Types

Bar Navigation

Bar Navigation are row-based navigation panels that align across your page commonly you'd see these at the top of a page.

Side Navigation

Side Navigation is column-based navigation that is positioned on the side of a page align from top to bottom. Commonly you'd see these in a tutorial website where they are separated into different page topics inside a side navigation pane.

Invisible Navigation

Hidden navigations are block-based navigations that involve the use of block values to display the link. Known as secondary navigation simply it extends sub links of one parent link. Commonly you'd see these types of navigation when you hover or click on a parent link.

Footer Navigation

Footer navigation are both row-based or column-based navigation types. This all depends on the site's design. When you scroll down to the bottom of a page, you'd see a variety of important links. These are commonly known to be Privacies, FAQs, Terms and Conditions.

Understanding Display

Don't Float Just Display

One of the main differences in why we could use Display instead of Float is that we can align our navigation responsive as possible into the center. We can neatly position the navigation links without it spreading across the page. No matter if your width shrinking because you can use media queries to resolve most sizing issues. Float tend to be difficult to align center depends on how developers can position them but it can make it most responsive to the right or left which can cause the navigation to be difficult to scale from one side. Using display which can center the links and makes it more responsive as it can be even when resizing. Some considerations are in place to mind are size and device. Always be aware of how you'd make your navigation responsive displaying all the navigation links on one page. There has to be a limit of links counted on a page.

Sometimes using float for your navigation can be fine in context but this all depends on how the navigation is expected to look and scaled like whether it is right-aligned or left-aligned. Also, when your navigation links are aligned centered it's easier to reconsider design side-by-side. This comes to having the space to improving a website's navigation structure by all means, it can become simply adjusted to suit your website's look and feel.

The top four values of the Display Property:

None

This value represents that whatever a block you have created will not be displayed because it's identified as none.

Inline-Block and Inline

These are both similar in how they are lined but differently based on their values given. This will display blocks in a row beside each other.

Block

Compared to the inline values, the block value doesn't align beside each other it creates a new row by laying blocks below each other as a column.

Display ceoncepts in navigations.

Avoid Using "block" For Bar Navigation

As explained about the block value, we don't use this method to create the bar navigation because remember it only builds the blocks from top to down. We can only use this if we wish to create a navigation height-wise i.e. a side content pane. However, you can use this to display an extended navigation column inside your main navigation.

Using Block in navigations.

Using "inline-block" Over "inline"

One of the reasons why we should use inline-block instead of inline is that it is more complex in configuration. If you wanted to avoid any overlaps even if you use padding or height, the inline-block will be more useful to your sturcture. Creating the Bar Navigation

Start by creating the HTML file for our navigation bar where we will use an id instead of class. Creating an id called nav-bar, will be our primary navigation bar for our website.

We will also include our hyperlinks under the <div id="nav-bar"> tag. Using the <a></a> tag as our links which will be used to align and structure together.

The sample code below is our first step of creating the navigation bar:
Styling the Nav Bar with CSS
<div id="nav-bar">   <a href="#home">Home</a>   <a href="#page1">Page 1</a>   <a href="#page2">Page 2</a>   <a href="#page3">Page 3</a>   <a href="#page4">Page 4</a> </div>

You can see it will just display blue underlined texts which means they are hyperlinks alined together, however, we do need to add some style to make it better.

Styling the Navigation

Adding style to our navigation bar will help make it interesting for the visitors to navigate on your website. You will also have understood the properties discussed at this stageThe understanding display values of the process.

Styling the Nav Bar with CSS
#nav-bar {   background-color: slateblue;   width: 100%;   height: 50px;   text-align: center; }
#nav-bar a {   background-color: darkslateblue;   /*background: rgba(0,0,0,0.5);*/   /* Uncomment the above line to test for overlaps */   color: white;   padding: 16px 10px;   font-family: Arial;   font-size: 16px;   font-weight: bold;   text-decoration: none; }

Note:

- If you wish to create it externally apply the link tag below the title tag inside your head tag.

- If you only wish to style internally then add the style tag inside the head tag and add your CSS code there.

Using Inline-Block Method

Remember, from this stageThe understanding display values explained about the differences between inline and inline-block. In this exmaple we'll use inline-block with padding values.

With the CSS you have already seen you can add to the end of the #nav-bar a { the display property with the code below.

Adding the Inline
#nav-bar {   background-color: slateblue;   width: 100%;   height: 50px;   text-align: center; }
#nav-bar a {   background-color: darkslateblue;   /*background: rgba(0,0,0,0.5);*/   /* Uncomment the above line to test for overlaps */   color: white;   padding: 16px 10px;   font-family: Arial;   font-size: 16px;   font-weight: bold;   text-decoration: none;   /* applying inline-block */   display: inline-block; }

The Inline Problem

One problem you might face is the adjustments of the blocks. Sometimes it can be resolved by just applying the margin-right with -4px. One of the problems that comes to mind is that based on fonts, sizing and links you may likely to see thin gaps or some overlaps.

To resolve this issue by applying a full margin value into margin: with 4 values involved. Remember 4 values represents top(T), right(R), bottom(B) and left(L) margin: T R B L;.

We don't need the top and bottom values so we remain them with 0 but we do focus on the sides. The right side the most, we add -2px to the left and add -2.5px. We focus on the right side adjustments when required.

Adjusting Margins
#nav-bar {   background-color: slateblue;   width: 100%;   height: 50px;   text-align: center; }
#nav-bar a {   background-color: darkslateblue;   /*background: rgba(0,0,0,0.5);*/   /* Uncomment the above line to test for overlaps */   color: white;   padding: 16px 10px;   font-family: Arial;   font-size: 16px;   font-weight: bold;   text-decoration: none;   display: inline-block;   /* If you notice gaps and overlaps on links adjust the right side margin */   margin: 0 -2px 0 -2.5px; }
